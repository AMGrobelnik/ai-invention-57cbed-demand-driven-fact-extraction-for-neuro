## Full Summary

Three benchmark datasets were acquired from HuggingFace and standardized to the exp_sel_data_out schema for use in a neuro-symbolic reasoning experiment that converts unstructured text to first-order logic (FOL) for symbolic Prolog execution. Total: 422,987 examples across 3 datasets.

**Dataset 1: CLUTRR_v1** (53,518 examples) — EMNLP 2019 benchmark for compositional language understanding via kinship multi-hop reasoning. Each example is a short story about family members; the model must infer the kinship relation between two named entities. Ground-truth proof_state field encodes the FOL reasoning chain explicitly (e.g., daughter-sister → niece). Hop depths 2–10. Key fields: input (story + query), output (kinship label), metadata_hop_depth, metadata_relation_chain, metadata_proof_state. Ideal for testing FOL relation composition in Prolog.

**Dataset 2: RuleTaker** (237,906 examples) — IJCAI 2020 benchmark (Clark et al.) for rule-based logical entailment. Each example gives NL facts+rules as context and a statement to verify (entailment / not entailment). Depth variants 1–5 encode the required inference chain length. Key fields: input (context + statement), output (entailment/not entailment), metadata_hop_depth, metadata_config. Directly maps to FOL rule parsing + Prolog forward chaining.

**Dataset 3: ProofWriter** (131,563 examples) — ACL 2021 extension of RuleTaker with explicit proof traces in allProofs field. Same logical entailment task but adds ground-truth proof derivations and NFact/NRule counts. Depth 0–5. Key fields: input (theory + question), output (True/False), metadata_n_facts, metadata_n_rules, metadata_hop_depth. Proof traces enable supervised training of the FOL translation step.

**Dropped:** RuleTaker_D5 (axon-rl/RuleTaker-d5-70k) — only covers depth-5 examples, no proof traces, redundant with RuleTaker's depth-5 coverage and inferior metadata.

**Output files:** full_data_out/ directory contains 5 split files (all under 100MB) totaling 329MB. mini_data_out.json (3 examples per dataset, 9 total) and preview_data_out.json (3 truncated examples per dataset) provided for inspection. Schema validated against exp_sel_data_out format. data.py is the reproducible loader with JSONL-fallback for malformed JSON arrays.
